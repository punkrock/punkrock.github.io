0. Requirements

I recommend a server with 2 or 4 CPU's and a minimum of 8 GB RAM for a reliable Service/API-node!

1. Add A-Records for api and service

Login to your domain hoster and set two A RECORDS for api and service.

This example will result in the domains:
- https://lisk-mainnet-api.YourDomain.tld
- https://lisk-mainnet-service.YourDomain.tld





2. Create user and install some stuff

~# adduser lisk

~# gpasswd -a lisk sudo

~# visudo

Add this line to the end of the file: 

lisk ALL=(ALL) NOPASSWD: ALL

~# su - lisk

~$ sudo apt-get -y update && sudo apt-get -y upgrade && sudo apt-get -y dist-upgrade && sudo apt-get -y install wget tar zip unzip ufw htop nano npm git curl bash jq screen && sudo apt-get -y install unattended-upgrades && sudo nano /etc/apt/apt.conf.d/20auto-upgrades

Add this to the file: 

APT::Periodic::Update-Package-Lists "1";
APT::Periodic::Download-Upgradeable-Packages "1";
APT::Periodic::AutocleanInterval "7";
APT::Periodic::Unattended-Upgrade "1";

Save & exit

~$ sudo apt-get install -y build-essential git make

~$ sudo apt install nginx -y



3. Secure SSH

~$ cd && mkdir .ssh && cd .ssh && ssh-keygen -o -a 100 -t ed25519

Always press ENTER to confirm

~$ cat id_ed25519.pub >> authorized_keys

~$ chmod 600 authorized_keys && cd .. && chmod 700 .ssh

Save the private key (id_XXXXXXX) to your hard drive.
Then delete the private key (id_XXXXXXX) and the public key (id_XXXXXXX.pub) on your server

~$ sudo nano /etc/ssh/sshd_config 

Port 24

# Authentication:
LoginGraceTime 120
PermitRootLogin no
StrictModes yes
AuthorizedKeysFile	%h/.ssh/authorized_keys
PermitEmptyPasswords no
ChallengeResponseAuthentication no
PasswordAuthentication no

Save & Exit

~$ sudo /etc/init.d/ssh restart

Now edit your SSH settings with your new port and keys and start a new session.



4. Config Firewall

~$ sudo ufw default deny
~$ sudo ufw allow 'Nginx Full'
~$ sudo ufw allow 24,8000,8001/tcp
~$ sudo ufw enable

Confirm with "yes"



5. Install Lisk Core

~$ wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.38.0/install.sh | bash

Now start a new SSH session!

~$ nvm install 16

~$ npm install --global --production lisk-core

~$ cd && nano core-start.json 

{
  "name": "Lisk Core",
  "script": "lisk-core start --enable-http-api-plugin --api-ws --api-ws-port=8080 --http-api-plugin-port=8000 --api-ws-host 0.0.0.0",
  "env": {
    "LISK_NETWORK": "mainnet"
  }
}



6. Install pm2

~$ sudo npm i -g pm2



7. Install MySQL Server

~$ sudo apt install mysql-server -y

~$ sudo mysql_secure_installation

First type "No" (secure password stuff) and choose a simple password like "123" (you will change it to a secure one in the next step). For the other questions type always "Yes".

Now connect to MySQL and change your root password and make sure to use at least 16 characters, lower case, upper case, numbers. Also tested and working are ! " @. Don't use % * = -

~$ sudo mysql -u root -p

CREATE DATABASE mainnet;
ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY 'YourNewCr4zypa55w0rd!!!"""@qwertz';
FLUSH PRIVILEGES;
\q

~$ sudo systemctl restart mysql

~$ sudo systemctl enable mysql

10. Config Nginx and get TLS certificates

~$ sudo nano /etc/nginx/sites-available/default

Now empty the file (CTRL + k) completely and add the following and make sure to change the URL's to your domain!: 

server {
    listen      80;
    server_name lisk-mainnet-api.YourDomain.tld;
}

server {
    listen       80;
    server_name lisk-mainnet-service.YourDomain.tld;
}

~$ sudo rm -rf /etc/nginx/sites-enabled/default
~$ sudo ln -s /etc/nginx/sites-available/default /etc/nginx/sites-enabled/
~$ sudo systemctl restart nginx
~$ sudo apt install python3-certbot-nginx -y
~$ sudo certbot --nginx -d lisk-mainnet-api.YourDomain.tld -d lisk-mainnet-service.YourDomain.tld

If asked for redirects, press 2 (redirect)

~$ sudo nano /etc/nginx/sites-enabled/default

Now empty the file (CTRL + k) completely and add the following and make sure to change the URL's to your domain!: 

server {
  listen 443 ssl;
  server_name lisk-mainnet-api.YourDomain.tld;
  ssl_certificate /etc/letsencrypt/live/lisk-mainnet-api.YourDomain.tld/fullchain.pem; # managed by Certbot
  ssl_certificate_key /etc/letsencrypt/live/lisk-mainnet-api.YourDomain.tld/privkey.pem; # managed by Certbot
  ssl_verify_client off;
  ssl_protocols TLSv1 TLSv1.1 TLSv1.2;

12. Add auto-start for Lisk Core and Lisk Service after reboot using pm2

Important infos:
It will now take some time until your node is in sync with the network, that's why we put 120 seconds sleep in our start-lisk-service.sh script.
It could be that it takes much longer, especially when the chain is bigger and you start from scratch. If you start Lisk Service before Lisk Core is in sync, it can lead to database errors.
If you started from scratch, only run Lisk Core (pm2 start core-start.json) and let it get in sync with the network and stop Lisk Core before you use this start-lisk-service.sh script.

Make PM2 ready for reboots:

~$ cd && pm2 startup

You got an output with a sudo command. You need to copy paste and execute it. PM2 is ready now for reboots.


Now start Lisk Core:

~$ cd && pm2 start core-start.json

Now create a script to start Lisk Service:

~$ cd && nano start-lisk-service.sh

#!/bin/bash
sleep 120
cd && cd lisk-service
pm2 start ecosystem.core3.config.js
sleep 5
pm2 delete start-lisk-service

Make it executable and start it:

~$ chmod +x start-lisk-service.sh
~$ cd && pm2 start start-lisk-service.sh

Now you need to enter the following command within 120 seconds:
~$ pm2 save

Now open the pm2-monitor. After about two minutes Lisk Service (core, gateway, market) should start and 5 seconds later the process "start-lisk-service" should disappear:

~$ pm2 monit

Now reboot, open pm2 monit and wait about 120 seconds to see if everything works and gets started as expected.

To see if everything works, you can visit the following URL's:
- https://lisk-mainnet-service.YOURDOMAIN.TLD/api/v2/network/status
- https://lisk-mainnet-service.YOURDOMAIN.TLD/api/status

If you see results loading and no error, you are finished.

If your node is in sync you can use Lisk Desktop to connect to your Service node -> https://lisk-mainnet-service.YOURDOMAIN.TLD



13. Manual Stop / Start procedure

If you wanna stop everything and start it manual use the follwing commands.

To stop Lisk Core and Lisk Service:

~$ pm2 delete all

To start Lisk Core and Lisk Service:

~$ cd && pm2 start core-start.json

~$ cd && cd lisk-service && pm2 start ecosystem.core3.config.js



14. Update Lisk Service

~$ sudo apt-get update && sudo apt-get upgrade && pm2 delete all && npm remove pm2 -g && npm install pm2 -g && pm2 update && pm2 --update-env
~$ pm2 startup

Now copy paste the output line from the previous command (pm2 startup)

~$ pm2 delete all
~$ cp lisk-service/ecosystem.core3.config.js ecosystem.core3.config.js
~$ rm -rf lisk-service
~$ npm remove -g lisk-service

~$ git clone https://github.com/LiskHQ/lisk-service.git
~$ cp ecosystem.core3.config.js lisk-service/ecosystem.core3.config.js
~$ rm -rf ecosystem.core3.config.js
~$ cd lisk-service
~$ git fetch && git pull
~$ git checkout version number
~$ make build-local
~$ cd && cd lisk-service/services/market
~$ npm i
Now start Lisk Core again and start the Lisk Service startup script:
~$ cd
~$ pm2 start core-start.json
~$ pm2 start start-lisk-service.sh
~$ pm2 monit

Now save the pm2 processes and execute the following command within 120 seconds:
~$ pm2 save

~$ sudo reboot
